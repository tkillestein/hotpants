# Delta Function Kernel Basis Implementation Plan

## Overview

Implement Bramich (2008) delta function kernel basis as a selectable, pluggable alternative to the current multi-Gaussian basis. Design the system to support future basis variants (PCA, Fourier, etc.) without monolithic changes.

**Goal:** Allow users to choose between basis representations via configuration, with proper regularization and spatial variation support.

**Reference:** [Bramich (2008)](https://arxiv.org/abs/0802.1273) "The Optimal Difference Image Combination of Dithered Images," MNRAS 389:1365

---

## Architecture Design

### 1. Pluggable Basis System

**Current state:** Kernel basis is hardcoded as multi-Gaussian via `getKernelVec()` and related functions.

**Proposed change:** Abstract basis representation into a pluggable interface.

**Key design decisions:**

1. **Basis abstraction (new):**
   - Define a `kernel_basis_t` enum and function pointers for basis operations
   - Support: `GAUSSIAN_BASIS` (existing), `DELTA_BASIS` (new), future types
   - Avoid touching existing Gaussian code path; add delta path alongside

2. **Configuration:**
   - Add `-basisType` CLI flag (default: `gaussian`, options: `gaussian`, `delta`)
   - Add `KernelConfig.basis_type` Python field (enum or string)
   - Add `-deltaKerGridSize` to specify delta grid spacing (pixels; e.g., 2, 3, 4)
   - Add `-deltaRegularization` for Laplacian smoothness penalty (e.g., 1e-3)

3. **Global state management:**
   - Extend `globals.h`/`globals.c` with:
     - `iBasisType` (kernel_basis_t enum)
     - `rDeltaGridSize` (double, in pixels)
     - `rDeltaRegularization` (double, smoothness penalty)
   - Ensure Python API (`_core.py`) manages these during `global_state()` context

### 2. Delta Function Basis Implementation (C Core)

**File: alard.c (new functions; ~600–800 lines)**

#### 2.1 RBF-based Delta Basis

Delta functions at grid points are impractical numerically. Instead, use a "softened" delta basis: narrow Gaussians or RBFs centered at grid points.

**Approach:** Use a 2D regular grid of basis functions with spacing `deltaKerGridSize` pixels. Each basis element is a narrow radial basis function (e.g., thin-plate spline or Gaussian with fixed narrow σ).

**Prototype functions:**

```c
/**
 * @brief Initialize delta function basis grid within kernel footprint.
 * @details Allocates basis functions on a regular 2D grid with spacing deltaKerGridSize.
 *          For a kernel of half-width hwKernel, computes grid points (cx, cy) and
 *          assigns basis indices. Stores grid geometry in global state for kernel
 *          evaluation and matrix building.
 * @param[out] nBasisFuncs Number of basis functions allocated
 * @return nBasisFuncs on success, -1 on memory error
 */
int init_delta_basis_grid(int *nBasisFuncs);

/**
 * @brief Evaluate delta basis function i at offset position (dx, dy) from grid center.
 * @details Returns the basis function value (RBF or narrow Gaussian) centered at grid
 *          point i, evaluated at pixel offset (dx, dy). Used by xy_conv_stamp_delta()
 *          and spatial_convolve_delta().
 * @param[in] basisIdx Index of basis function (0 to nBasisFuncs-1)
 * @param[in] dx X offset in pixels
 * @param[in] dy Y offset in pixels
 * @return Basis function value
 */
double eval_delta_basis(int basisIdx, double dx, double dy);

/**
 * @brief Convolve image stamp with delta basis function i.
 * @details Equivalent to xy_conv_stamp() for Gaussian basis; computes correlation
 *          of stamp with basis function i over the stamp region.
 * @param[in] stamp Input image stamp
 * @param[in] mStampX, mStampY Stamp dimensions
 * @param[in] basisIdx Index of delta basis function (0 to nBasisFuncs-1)
 * @param[out] pixFitted Accumulator for convolution result
 * @return 0 on success, -1 on error
 */
int xy_conv_stamp_delta(double *stamp, int mStampX, int mStampY, int basisIdx, double *pixFitted);

/**
 * @brief Build normal equation matrix row for delta basis.
 * @details Accumulates Aᵀ·A and Aᵀ·b for least-squares fit of delta coefficients
 *          on a single stamp. Called by fitKernel() when basis_type == DELTA_BASIS.
 * @param[in] substampIdx Index of substamp (bright star center)
 * @param[in,out] kernelSol Coefficient array (updated with accumulated equations)
 * @param[in,out] iMatrixSize Dimension of normal equation matrix (may change)
 * @return 0 on success, -1 on error
 */
int build_matrix_delta(int substampIdx, double *kernelSol, int *iMatrixSize);
```

#### 2.2 Regularization (Laplacian Smoothness)

Add L2 penalty on kernel Laplacian to prevent overfitting and noise amplification.

**Approach:** After accumulating normal equations Aᵀ·A and Aᵀ·b, add a regularization matrix:

```
min ||A·x - b||² + λ·||L·x||²
```

where L is the discrete Laplacian operator on the 2D delta grid and λ = `deltaRegularization`.

**Prototype functions:**

```c
/**
 * @brief Assemble Laplacian regularization matrix for delta basis grid.
 * @details Builds the discrete 2D Laplacian (5-point stencil) as a matrix operator
 *          on the delta grid. Returns L·L^T scaled by regularization weight.
 * @param[in] nBasisFuncs Number of delta basis functions
 * @param[in] gridWidth, gridHeight Grid dimensions (nx, ny)
 * @param[in] regularizationWeight λ parameter
 * @param[out] laplacianRegularization Output regularization matrix (nBasisFuncs x nBasisFuncs)
 * @return 0 on success, -1 on memory error
 */
int assemble_laplacian_regularization(
    int nBasisFuncs, int gridWidth, int gridHeight,
    double regularizationWeight, double *laplacianRegularization);

/**
 * @brief Apply Laplacian regularization to normal equation matrix.
 * @details Adds regularization matrix to accumulated Aᵀ·A before LAPACK solve.
 * @param[in,out] matrix Input normal equation matrix (modified in place)
 * @param[in] matrixSize Dimension of matrix
 * @param[in] regularization Regularization matrix
 * @return 0 on success
 */
int apply_regularization(double *matrix, int matrixSize, const double *regularization);
```

#### 2.3 Kernel Evaluation

Support evaluation of delta-based kernel at arbitrary positions for convolution.

```c
/**
 * @brief Evaluate spatially-varying kernel using delta basis at position (xi, yi).
 * @details For polynomial or TPS spatial variation, coefficients cᵢ(x,y) multiply
 *          delta basis functions. This function evaluates the kernel at pixel (xi, yi).
 * @param[in] xi, yi Image coordinates
 * @param[in] kernelSol Kernel solution coefficients (delta basis weights or spatial params)
 * @param[out] kernel Kernel array to fill (hwKernel × hwKernel)
 * @return 0 on success
 */
int make_kernel_delta(double xi, double yi, const double *kernelSol, double *kernel);

/**
 * @brief Spatially-varying convolution using delta basis kernel.
 * @details Parallel to spatial_convolve_fft() and spatial_convolve().
 *          Applies FFT-accelerated convolution with spatially-varying delta kernel.
 * @param[in] image Template image
 * @param[in,out] diffimage Difference image (output)
 * @param[in] kernelSol Delta basis kernel solution
 * @return 0 on success
 */
int spatial_convolve_delta(double *image, int mImageX, int mImageY,
                           double *diffimage, const double *kernelSol);
```

### 3. Integration Points (Existing Code Paths)

#### 3.1 `getKernelVec()` dispatcher

Current: Hardcoded Gaussian basis initialization.

Change: Add basis-type dispatch:

```c
int getKernelVec(int *nCompKer) {
    switch(iBasisType) {
        case GAUSSIAN_BASIS:
            return getKernelVec_gaussian(nCompKer);
        case DELTA_BASIS:
            return init_delta_basis_grid(nCompKer);  // nCompKer = nBasisFuncs
        default:
            LOG_ERROR("Unknown basis type");
            return -1;
    }
}
```

#### 3.2 `xy_conv_stamp()` dispatcher

Current: Gaussian-specific separable convolution.

Change: Add dispatcher:

```c
int xy_conv_stamp(..., int basisIdx, ...) {
    switch(iBasisType) {
        case GAUSSIAN_BASIS:
            return xy_conv_stamp_gaussian(...);
        case DELTA_BASIS:
            return xy_conv_stamp_delta(..., basisIdx, ...);
        // ...
    }
}
```

#### 3.3 `fitKernel()` dispatcher

Current: Calls `build_matrix()` for all basis functions.

Change: Add basis-aware path:

```c
// In fitKernel():
if(iBasisType == DELTA_BASIS) {
    // build_matrix_delta accumulates Aᵀ·A and Aᵀ·b
    for each substamp {
        build_matrix_delta(substampIdx, kernelSol, &iMatrixSize);
    }
    // Apply Laplacian regularization
    assemble_laplacian_regularization(...);
    apply_regularization(kernelSol, iMatrixSize, laplacianReg);
}
// Shared LAPACK solve for both bases
dpotrf_(...);
dpotrs_(...);
```

#### 3.4 `make_kernel()` dispatcher

Current: Evaluates polynomial-weighted Gaussian sum.

Change: Add basis dispatcher:

```c
int make_kernel(double xi, double yi, const double *kernelSol, double *kernel) {
    switch(iBasisType) {
        case GAUSSIAN_BASIS:
            return make_kernel_gaussian(...);
        case DELTA_BASIS:
            return make_kernel_delta(...);
        case TPS:  // Could apply to both bases
            return make_kernel_tps(...);
    }
}
```

#### 3.5 `spatial_convolve()` dispatcher

Current: Chooses between direct and FFT convolution.

Change: Add basis selector:

```c
int spatial_convolve(...) {
    switch(iBasisType) {
        case GAUSSIAN_BASIS:
            return spatial_convolve_fft(...);  // Existing
        case DELTA_BASIS:
            return spatial_convolve_delta(...);
        // ...
    }
```

## Key Design Decisions & Rationale

### 1. Basis Abstraction via Dispatcher (not OOP)

**Decision:** Use function pointers / switch statements instead of C struct-based polymorphism.

**Rationale:**
- Simpler implementation; avoids struct overhead
- Minimal changes to existing Gaussian code path
- Easier to debug; explicit control flow
- C idiom (avoids C++ OOP patterns that are awkward in C)

### 2. Delta Functions as Narrow RBFs

**Decision:** Don't use true delta functions (δ(x - xᵢ)); instead use narrow Gaussians or thin-plate splines.

**Rationale:**
- True delta functions are non-smooth; numerically unstable
- Narrow RBFs approximate delta behavior while being stable
- Allow user to adjust grid spacing and regularization for control
- Align with TPS basis already implemented

### 3. Laplacian Regularization (not Tikhonov)

**Decision:** Use discrete 2D Laplacian penalty instead of simpler L2 Tikhonov.

**Rationale:**
- Specifically penalizes curvature, not all variation
- Bramich (2008) uses Laplacian; aligns with literature
- Encourages smooth kernels while allowing necessary features
- More interpretable regularization parameter

### 4. No Spatial Variation Dispatch (Reuse Polynomial/TPS)

**Decision:** Delta basis reuses existing polynomial and TPS spatial variation; no separate `make_kernel_delta_tps()`.

**Rationale:**
- Spatial variation is orthogonal to basis choice
- Reduces code duplication
- Dispatcher in `make_kernel()` handles combination
- Simplifies testing and maintenance

### 5. FFT-based Convolution for Delta (Parallel to Gaussian)

**Decision:** Delta basis uses FFT convolution like Gaussian (not direct convolution).

**Rationale:**
- Mandatory FFT per CLAUDE.md design
- Delta kernels can be large (many basis functions); FFT necessary for performance
- Aligns with existing architecture
- Avoids code path bifurcation (direct vs. FFT)

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Regularization tuning difficult | Delta basis unusable | Provide sensible defaults; document parameter sweep methodology |
| Delta basis slower than Gaussian | Performance regression | Profile early (Phase 5); optimize hot paths if needed |
| Grid geometry edge cases | Subtle bugs | Unit tests for grid with hwKernel close to grid spacing |
| Interaction with TPS spatial variation | Unexpected behavior | Separate test suite for delta + TPS combination |
| Global state management errors | Silent failures | Validate globals in `global_state()` context; add assertions |
